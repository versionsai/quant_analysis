# Futu Trade Skill

本 Skill 用于说明 Futu OpenAPI 的交易能力边界，以及它在本项目中的适用范围。

## 先说结论

- 富途官方 `openapi` skill 覆盖交易能力
- 但本项目当前代码 **没有接入 Futu 实盘交易**
- 项目当前交易链路仍以模拟交易为主

因此：
- 可以把本 Skill 用作“交易接入前置清单”
- 不应把它理解为“本项目已经支持实盘下单”

## 官方能力边界

根据富途官方 AI / Skills 文档：

- `openapi` skill 覆盖行情查询和交易操作
- 交易默认使用模拟环境
- 如果切换到实盘交易，需要显式说明并二次确认
- 使用时需要关注权限、交易环境、限频、账户解锁等前置条件

## 本项目当前状态

当前项目中的交易实现主要是：
- `trading/broker.py`
- `trading/orders.py`
- `trading/simulate_trading.py`
- `trading/recommend_recorder.py`

当前特点：
- 已有模拟下单、订单对象、成交记录
- 已有持仓、止盈止损、风控和自动卖出逻辑
- 尚未接入 Futu 的真实交易上下文

换句话说：
- 当前项目适合“信号 -> 推荐 -> 模拟买卖 -> 复盘”
- 还不适合“信号 -> Futu 实盘下单”

## 如果未来接入 Futu 交易，需要满足的条件

### 1. 环境侧

- OpenD 已安装并登录
- 账户具备对应市场交易权限
- 明确使用模拟盘还是实盘
- 本机/服务器可访问 OpenD

### 2. 代码侧

至少需要新增或封装这些能力：
- 交易上下文创建
- 交易环境切换
- 账户解锁
- 下单
- 撤单
- 查询持仓
- 查询订单状态
- 查询资金

### 3. 风控侧

必须补齐：
- 实盘交易前二次确认
- 最大下单金额限制
- 市场状态检查
- 下单失败重试与幂等保护
- 风险日志和审计记录

## 建议的接入顺序

1. 先接入模拟盘交易上下文
2. 做只读接口：
   - 账户
   - 持仓
   - 订单查询
3. 再做小额模拟盘下单
4. 最后才考虑实盘

## 不建议直接做的事

- 不建议让 AI 直接无确认发起实盘下单
- 不建议在没有解锁、环境确认、风控兜底前接实盘
- 不建议把“行情可用”误认为“交易链路可用”

## 和本项目的关系

本项目当前最适合先做的是：
- 用 Futu 补强行情层
- 用五档盘口增强盘中判断
- 保持交易层仍然是模拟执行

如果未来确定接 Futu 交易，建议新增：
- `trading/futu_broker.py`

由它负责：
- Futu 交易上下文
- 模拟盘/实盘环境隔离
- 下单与撤单封装
- 和现有 `Broker` 接口对齐
